<objective>

Build a complete narrative from a raw idea, logline, or premise through to a scored,
diagnostically-verified draft. The workflow moves through seven phases: intake, hamartia
definition, framework selection, character assembly, architecture planning, draft
generation, and self-diagnostic review.

</objective>

<mode_awareness>

This workflow respects the 3-mode system. The **current mode** shapes behavior at every step.

- **Brainstorm Q&A (DEFAULT):** Ask 2–4 targeted questions per round. Do NOT generate
  narrative text until the user signals readiness ("go", "build it", "draft it").
- **Collaborative:** Generate in chunks (scene, chapter, act). Pause after each chunk for
  feedback. Incorporate feedback immediately.
- **Fast-Auto:** Use whatever context is available. Infer missing details. Generate the
  complete output in one pass, then run diagnostics and present results.

**The user can switch modes at any point.** Watch for:
- "Let's brainstorm" / "ask me questions" → Brainstorm Q&A
- "Let's collaborate" / "work with me" → Collaborative
- "Just go" / "auto mode" / "fast mode" → Fast-Auto
- "Pause" / "stop" → Halt and await direction

</mode_awareness>

<process>

**Step 1 — Mode Check**

Acknowledge the current operating mode. If this is the first interaction and the user has
not specified a mode, default to Brainstorm Q&A.

State: *"I'm in [Mode Name] mode. You can switch at any time by saying 'let's brainstorm',
'let's collaborate', or 'just go'."*

---

**Step 2 — Intake Interview**

Gather the foundational parameters for the story:

- **Logline or premise** — What is this story about in one sentence?
- **Genre** — Literary fiction, thriller, sci-fi, fantasy, horror, romance, etc.
- **Tone** — Dark, comedic, lyrical, sparse, satirical, etc.
- **Target audience** — Age range, sophistication level, genre expectations
- **Approximate length** — Flash fiction, short story, novella, novel chapter(s)

*Mode behavior:*
- **Brainstorm Q&A:** Ask these as conversational questions, 2–3 at a time. Synthesize
  answers before proceeding. Probe for specificity — reject vague answers gently.
- **Collaborative:** Present a brief intake checklist. Let the user fill what they want.
  Infer the rest and present assumptions for approval.
- **Fast-Auto:** Extract whatever is available from the user's initial message. Infer all
  missing parameters based on genre conventions and state your assumptions.

---

**Step 3 — Hamartia Definition**

Reference: `references/03-archetypal-database.md`

Define the protagonist's psychological core:

1. **Psychological flaw (hamartia)** — The deep error in judgment or perception that drives
   their behavior. This is NOT a quirky weakness — it is a fundamental misunderstanding of
   the world that causes real harm.

2. **Moral weakness** — Following Truby's framework, explicitly define HOW the protagonist's
   flaw is actively hurting others. The flaw must generate collateral damage to allies or
   the environment.

3. **The key question to ask (or answer in Fast-Auto):**
   *"How is this flaw hurting others right now? Who specifically is being damaged, and how?"*

*Mode behavior:*
- **Brainstorm Q&A:** Present the concept of hamartia. Ask the user to describe their
  protagonist's deepest flaw. Follow up with: "How is this flaw hurting the people around
  them?" Push for specificity.
- **Collaborative:** Propose 2–3 hamartia options based on the logline and genre. Present
  each with its implied moral weakness. Let the user select or modify.
- **Fast-Auto:** Derive the hamartia from the logline. State the moral weakness explicitly.
  Proceed without stopping.

---

**Step 4 — Framework Selection**

Reference: `references/02-structural-paradigms.md`

Based on the story's Designing Principle (the central idea that governs the narrative's
architecture), select the structural paradigm:

| Story Driver | Recommended Framework |
|---|---|
| Protagonist's internal flaw drives the plot | **Truby's Organic Architecture** |
| Multiple perspectives, objective/subjective split | **Dramatica Theory** |
| Emotional resonance without adversarial combat | **Kishōtenketsu** |
| Relentless tension, thriller pacing | **Fichtean Curve** |

**Never default to "Save the Cat!" or generic three-act structure.**

Anti-pathology checks (apply regardless of framework):
- Mandate nonlinearity — use time jumps, flashbacks, interwoven subplots
- Enforce subplot integration — break single-track causal chains
- Embrace ambiguous endings — do not force tidy protagonist-solved resolutions

*Mode behavior:*
- **Brainstorm Q&A:** Ask diagnostic questions: "Is this story driven by the character's
  inner flaw? Or by external escalating crises? Do you want multiple perspectives?" Use
  answers to recommend a framework with full rationale.
- **Collaborative:** Present the framework table. Recommend one with rationale. Offer an
  alternative. Wait for the user's choice.
- **Fast-Auto:** Select the framework that best matches the logline and hamartia. State the
  selection and rationale. Proceed.

---

**Step 5 — Character Assembly**

References: `references/03-archetypal-database.md`, `references/04-agency-enforcement.md`
Template: `templates/character-profile.md`

Build complete character profiles:

1. **Protagonist**
   - Jungian Archetype (from the 12 archetypes)
   - Hamartia (from Step 3)
   - Shadow (the repressed, hidden aspect of their personality)
   - Aristotelian Ethos
   - Skills, knowledge, capabilities (Living Index)
   - Individuation state (where are they on the psychological journey?)
   - Voice notes and dialogue patterns

2. **Antagonist (Shadow-Matched)**
   - Must compete for the SAME desire as the protagonist
   - Must represent CONFLICTING moral values
   - Must externalize the protagonist's repressed Shadow
   - Cannot be a random villain — must be psychologically tethered to the protagonist

3. **Supporting Cast** (as needed)
   - Each with an archetype, a function in the story, and a relationship to the protagonist

*Mode behavior:*
- **Brainstorm Q&A:** Walk through the protagonist first. Ask archetype questions. Then
  construct the antagonist as a Shadow-match, explaining the logic. Ask for approval.
- **Collaborative:** Generate the protagonist profile. Pause. Then the antagonist. Pause.
  Incorporate feedback at each step.
- **Fast-Auto:** Generate all character profiles in one pass using the template.

---

**Step 6 — Architecture Brief**

References: `references/04-agency-enforcement.md`, `references/01-neurochemical-engine.md`
Template: `templates/story-architecture-brief.md`

Produce the story architecture document:

1. **Structural skeleton** — Act/beat breakdown specific to the chosen framework
2. **Agency enforcement checkpoints** — Mark which beats require protagonist
   Want→Decision→Action→Consequence. Mark which beats are authorized for atmospheric
   immersion, comedic relief, or reflection (Micro Spontaneity).
3. **Neurochemical pacing targets per act:**
   - Define target Cortisol/Oxytocin/Dopamine levels for each major section
   - Opening: Cortisol spike (disruption to status quo)
   - Middle: Oxytocin build (vulnerability, bonding)
   - Climax: All three elevated (Optimal Transportation)
   - Resolution: Dopamine release (earned catharsis)
4. **Foreshadowing map** — What must be planted early for the climax to feel earned
5. **Nonlinearity plan** — Where will time jumps, flashbacks, or parallel threads occur

*Mode behavior:*
- **Brainstorm Q&A:** Walk through each section of the architecture brief, asking for input.
  Present the neurochemical targets and ask if the user wants to adjust the emotional arc.
- **Collaborative:** Generate the full architecture brief. Pause for review. Adjust based on
  feedback.
- **Fast-Auto:** Generate the complete architecture brief in one pass. Proceed to drafting.

---

**Step 7 — Draft Generation**

Apply ALL essential principles throughout the draft:

1. **Logprob Override** — Reject the first, most obvious resolution. Choose the harder path.
2. **Mandatory Nonlinearity** — Use temporal complexity (flashbacks, time jumps, interwoven
   subplots). Never default to single-track chronological sequence.
3. **Anti-Moralization** — NEVER explicitly state the theme. The audience infers meaning
   through character action, consequence, and catharsis. (AI states themes explicitly 77%
   of the time — actively resist this.)
4. **Emotional Precision** — Balance somatic metaphors with explicitly named emotions. Cap
   the density of bodily metaphors per scene.
5. **Earned Catharsis** — Transformation exacts severe, irreversible cost. No deus ex machina.
   Coincidences into trouble: yes. Coincidences out of trouble: never.
6. **Real-World Engagement** — Reference specific names, brands, places, cultural touchstones.
   (AI uses vague allusions 72% of the time — actively resist this.)

*Mode behavior:*
- **Brainstorm Q&A:** This phase typically triggers a mode shift. Ask: "Ready for me to
  start drafting? I can work scene-by-scene (Collaborative) or generate the full draft
  (Fast-Auto). Or keep brainstorming."
- **Collaborative:** Generate scene-by-scene or chapter-by-chapter. After each chunk:
  - Present the chunk
  - State the neurochemical targets for that section and whether they were met
  - Ask: "Keep going? Adjust? Rewrite?"
  - Incorporate feedback into the next chunk
- **Fast-Auto:** Generate the complete draft in one pass. Apply all principles. Proceed
  directly to self-diagnostic.

---

**Step 8 — Self-Diagnostic**

References: `references/01-neurochemical-engine.md`, `references/05-narrative-diagnostics.md`,
`references/storyscope-anti-patterns.md`

Run the full diagnostic suite on the generated draft:

1. **Neurochemical Scoring** — Score each major scene on the three axes (Cortisol, Oxytocin,
   Dopamine) from 1–10. Identify the diagnostic profile for each:
   - Flatline (all < 4)
   - Stress Without Empathy (Cortisol > 7, Oxytocin < 4)
   - Pleasant but Stagnant (Oxytocin > 7, Cortisol < 4)
   - Action Without Soul (Cortisol & Dopamine > 7, Oxytocin < 4)
   - **Optimal Transportation (all > 7)** — the target state

2. **Pathology Scan** — Check for:
   - Continuity errors (character knowledge gaps, rule breaking, temporal conflicts)
   - Foreshadowing gaps (Chekhov's Gun violations, unearned twists)
   - Thematic over-explaining (narrator stating the moral)
   - Metaphor exhaustion (excessive somatic metaphors)
   - Vague allusions (generic worldbuilding instead of specific references)

3. **Agency Enforcement** — Verify:
   - Protagonist drives macro plot through Want→Decision→Action→Consequence
   - No false activity (character central but not active)
   - Irreversible consequences for key decisions
   - No deus ex machina or coincidence-based rescue
   - Does NOT force 100% protagonist-solved resolution (allow external fates)

4. **Verdict:**
   - If ANY diagnostic profile is NOT "Optimal Transportation", identify the specific issues
   - Generate 3–5 actionable rewrite directives targeting the lowest-scoring axes

*Mode behavior:*
- **Collaborative:** Present the full diagnostic report. Highlight the weakest sections.
  Ask: "Want me to revise these sections? Or do you want to address them yourself?"
- **Fast-Auto:** Auto-revise any section that does not score Optimal Transportation. Re-score
  after revision. Present the final draft with the before/after diagnostic summary.

</process>

<success_criteria>

The workflow is complete when ALL of the following are true:

- [ ] The story has a defined hamartia with an explicit moral weakness ("How is this flaw hurting others?")
- [ ] A structural framework has been consciously selected (not defaulted to generic three-act)
- [ ] All major characters have completed profiles with Jungian anchors and Shadow definitions
- [ ] The antagonist is Shadow-matched to the protagonist (same desire, conflicting values)
- [ ] The architecture brief includes neurochemical pacing targets per act
- [ ] The draft applies all 6 essential principles (Logprob Override, Nonlinearity, Anti-Moralization, Emotional Precision, Earned Catharsis, Real-World Engagement)
- [ ] The self-diagnostic has been run and all flagged issues have been addressed or acknowledged
- [ ] No scene relies on deus ex machina or coincidence-based rescue
- [ ] The theme is NEVER explicitly stated by the narrator

</success_criteria>
